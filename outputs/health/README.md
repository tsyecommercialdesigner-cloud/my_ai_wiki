# 健康检查（health check）说明

健康检查用来定期体检知识库，发现"断链、缺来源、信息过期、目录与标签不一致"等问题，
保证知识库长期可用、可追溯。建议每次集中整理资料后，或每隔一段时间，跑一轮检查。

检查结果写入 `outputs/health/` 下，按日期命名，例如 `outputs/health/2026-06-29.md`，
并在 `wiki/log.md` 里留一条记录。

## 检查清单

每轮健康检查至少覆盖以下七项：

1. **哪些 raw 资料没有来源摘要页**
   - 遍历 `raw/` 下的资料，确认每一份在 `wiki/sources/` 都有对应的来源摘要页。
   - 缺失的，补建来源摘要页。

2. **哪些页面缺来源**
   - 检查概念页、实体页、综合页里的重要判断，是否都指向了来源摘要页或原始资料。
   - 缺来源的判断，补链接，或标注"待核验"。

3. **哪些概念重复或冲突**
   - 查找 `wiki/concepts/` 下是否存在同义或近义的重复条目。
   - 查找不同页面对同一概念的描述是否相互矛盾。
   - 重复的考虑合并，冲突的标注并核验。

4. **哪些页面没有链接**
   - 找出"孤岛页面"：既没有链入、也没有链出的页面。
   - 为其补充关联（概念、实体、来源、综合之间的双链）。

5. **哪些判断可能过期**
   - 检查带时间性的结论（版本号、排名、价格、"最新/目前"等表述）。
   - 可能过期的，标注"待核验"并注明录入日期。

6. **哪些 raw 文件的目录和标签不一致**
   - `raw/articles/` 应使用 `article` 标签（微信公众号文章追加 `wechat`）。
   - `raw/clips/` 应使用 `clip` 标签。
   - `raw/papers/` 应使用 `paper` 标签。
   - 不得使用 `clippings` 作为 raw 类型标签。
   - 目录与标签对不上的，修正标签或移动文件。

7. **（可选）frontmatter 完整性**
   - 来源摘要页字段是否齐全：title、source、source_url、created、summary、key_points、related_concepts。
   - 缺字段的补齐，无法补齐的标注"待补充"。

## 检查报告模板

```markdown
# 健康检查报告 YYYY-MM-DD

## 概况
- 检查范围：……
- 总体结论：……

## 问题清单
| 类别 | 文件 | 问题 | 处理建议 |
| --- | --- | --- | --- |
| 缺来源摘要 | raw/... | …… | 补建 sources 页 |
| 缺来源 | wiki/... | …… | 补链接 |
| 概念重复/冲突 | wiki/concepts/... | …… | 合并 / 核验 |
| 孤岛页面 | wiki/... | …… | 补关联 |
| 可能过期 | wiki/... | …… | 标注待核验 |
| 目录标签不一致 | raw/... | …… | 修正标签 |

## 后续行动
- [ ] ……
```
