# 来源摘要页（sources）说明

每一份进入 `raw/` 的原始资料，都应在 `wiki/sources/` 下对应一张来源摘要页。
来源摘要页是知识库里所有判断"指向来源"的落脚点：概念页、实体页、综合页在引用观点时，都链接到这里。

## 命名约定

- 文件名与原始资料保持可对应，例如 `raw/articles/现有智能体框架分析.md` → `wiki/sources/现有智能体框架分析.md`。
- 一份原始资料对应一张来源摘要页。

## 字段模板

新建来源摘要页时，复制下面的 frontmatter 与正文骨架：

```markdown
---
title: 资料标题
source: 来源（书名 / 公众号名 / 网站名 / 作者等）
source_url: 原文链接（没有就写"待补充"）
created: YYYY-MM-DD
summary: 一两句话概括这份资料讲了什么
key_points:
  - 要点一
  - 要点二
  - 要点三
related_concepts:
  - "[[概念页A]]"
  - "[[实体页B]]"
---

# 资料标题

## 摘要

（用几句话说明这份资料的主题、范围、核心结论。）

## 关键要点

- 要点一
- 要点二

## 关联条目

- 概念：[[概念页A]]
- 实体：[[实体页B]]
- 综合：[[综合页C]]

## 原文位置

- 原始文件：`raw/.../文件名.md`
- 页码 / 章节：……
```

## 字段含义

| 字段 | 含义 |
| --- | --- |
| title | 资料标题 |
| source | 资料出处：书名、公众号名、网站、作者等 |
| source_url | 原文网络链接；纸质或扫描资料无链接时填"待补充" |
| created | 录入日期（YYYY-MM-DD） |
| summary | 一两句话的内容概括 |
| key_points | 这份资料的核心要点列表 |
| related_concepts | 关联的概念页 / 实体页，用 `[[双链]]` 指向 |

## 填写规则

- 不确定或原文缺失的信息，标注"待核验"或"待补充"，不要编造。
- 重要判断尽量在正文中指明来自原文哪一节 / 哪一页。
- 生成来源摘要页后，再去更新概念页、实体页、综合页，以及 `wiki/index.md` 和 `wiki/log.md`。
