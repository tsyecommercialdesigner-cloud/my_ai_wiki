# 扣子智能体构建平台：功能、能力与应用

## 摘要

本资料系统介绍字节跳动推出的一体化智能体构建平台「扣子（Coze）」，对应原文第 4 章。
内容涵盖平台定位与设计理念（4.1）、核心功能模块（4.2）、可实现的应用类型（4.3）与总结（4.4）。
扣子以低门槛、可视化、可扩展为目标，强调从"语言模型"向"可行动智能体"演进，
通过"知识库 + 工具系统 + 工作流引擎 + 多生态发布"的组合能力，支撑从知识服务到业务自动化的多类型智能体构建。

## 关键要点

### 平台定位与理念（4.1）

- **定位**：字节跳动一体化智能体构建平台，面向个人创作者、企业开发者，以及业务自动化、客服知识库、内容分发、企业机器人等场景。
- **四大核心理念**：
  1. 自然语言即编程 — 用自然语言/提示词定义智能体角色与行为，对应提示工程范式。
  2. 插件化工具调用 — 内置与自定义工具，让智能体从"生成文本"到"执行真实操作"。
  3. 统一工作流引擎 — 可视化流程编辑，构建多步骤、多条件、多工具协同的复杂任务。
  4. 多生态发布 — 发布到扣子商店、飞书、抖音、巨量引擎，或通过 API 嵌入第三方系统。

### 五大核心功能模块（4.2）

1. **智能体创建** — 人设配置、系统提示词、行为约束与边界、输出格式规范，相当于策略初始化（Policy Initialization）。
2. **知识库** — 检索增强（RAG），支持 PDF/Word/PPT/EPUB、SOP/FAQ、Markdown、URL、Excel/CSV 等知识源，自动解析、清洗、向量化索引。
3. **智能体技能（工具与插件系统）** — 联网搜索、调用 API、执行代码、数据库读写、图像类工具（OCR/PDF 解析）、第三方插件，实现"能回答"到"能行动"的跃迁。
4. **工作流** — 可视化自动化编排引擎，支持多步骤编排、条件分支、多工具串联、循环迭代、输入输出节点。
5. **发布与生态集成** — 扣子商店、飞书、抖音、巨量引擎、外部网站/API，形成"开发—编排—部署—运营"闭环。

### 五类智能体应用（4.3）

- **知识型**：以知识获取利用为核心，依赖知识库、RAG、提示工程，适用产品问答、专业咨询、文档检索、企业 FAQ。
- **工具型**：主动调用外部能力执行真实操作（数据库、Webhook、Python、外部 API），适用自动化办公、数据处理、系统联动。
- **创意型**：内容生成与创意表达（文案、分镜、PPT、绘图），结合豆包、Stable Diffusion、DALL·E 等。
- **工作流型**：流程执行与任务编排，接近企业级自动化机器人（合同审核、审批报告、直播评论处理）。
- **服务型**：面向真实用户的交互服务（客服、培训、营销），强调稳定性、一致性、可控性、安全合规。

### 总结（4.4）

扣子不仅是智能体创建工具，更是集开发、编排、部署与运营于一体的平台：降低开发门槛、增强专业问答、扩展行动能力、自动化复杂流程、推动从开发走向实际应用。
